# 🎬 Java 高并发电影票务管理系统

本项目基于 Spring Boot 3.x 全家桶与分布式中间件构建，是一个前后端分离的企业级 Web 应用。核心目标是解决电影票务系统在热门档期面临的高并发、超卖、数据一致性等挑战。

## ✨ 核心亮点

| 领域 | 难点解决 | 采用技术 | 效果 |
| :--- | :--- | :--- | :--- |
| **高并发** | 解决多人抢购同一座位导致的**“超卖”**问题。 | **Redis 分布式锁 (Lua 脚本) + MySQL 事务** | 确保原子锁座，彻底杜绝超卖，系统支持高并发请求。 |
| **异步解耦** | 解决订单超时未支付的实时处理和库存回滚。 | **RabbitMQ 死信队列 (DLX) + TTL 机制** | 实现订单 15 分钟未支付自动关单并释放库存，不占用数据库资源。|
| **数据检索** | 解决海量电影数据模糊查询效率低的问题。 | **Elasticsearch 7.x + IK 中文分词器** | 实现毫秒级的**全文检索**，支持电影名、演员、简介混合搜索及关键词高亮。|
| **架构优化** | 解决非结构化数据 (图片) 存储与主库耦合问题。 | **MinIO 对象存储** | 将海报、头像等图片与主库分离，提升 IO 性能及系统扩展性。|

## 🛠️ 技术栈 (Tech Stack)

| 分类 | 技术 | 版本 | 作用 |
| :--- | :--- | :--- | :--- |
| **后端核心** | Spring Boot | 3.2.5 | 快速开发框架，业务逻辑层实现。 |
| **数据库** | MySQL | 8.0 | 持久化存储，利用 JSON 字段灵活配置影厅座位。 |
| **缓存/锁** | Redis | 7.17.10 | 分布式锁、热点数据缓存 。 |
| **消息队列** | RabbitMQ | Latest | 异步解耦、订单超时处理。 |
| **搜索引擎** | Elasticsearch | 7.17.10 | 全文检索、数据聚合。 |
| **前端** | Vue 3, Vite, Element Plus | 最新版 | 现代化前端框架，实现可视化选座界面。 |

## ⚙️ 部署与运行 (Run Project)

本项目采用 **Docker Compose** 进行一键部署

1. **环境准备：** 确保已安装 Docker 和 Docker Compose。
2. **下载代码：** `git clone [你的仓库链接]`
3. **启动中间件：** 进入 `movie-middleware` 目录，执行 `docker-compose up -d` 启动 MySQL, Redis, ES, MinIO, RabbitMQ。
4. **启动后端：** 点击application文件上方的run按钮
5. **启动前端：** npm run dev
